Annotating Kubernetes Services for Humans
=========================================

This page outlines a convention for using annotations to help developers manage Kubernetes services. These annotations can be applied to a Kubernetes service with kubectl annotate. For example:

kubectl annotate service MYSERVICE a8r.io/owner=”sallydev”

Annotation Description
a8r.io/description Unstructured text description of the service for humans.
a8r.io/owner GitHub or equivalent username (prefix with @), email address, or unstructured owner description.
